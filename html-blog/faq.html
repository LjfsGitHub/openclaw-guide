📋 概述

本文档收集了OpenClaw使用过程中最常见的問題和解决方案。如果您遇到问题，请先查看这里，很可能已经有人遇到过并找到了解决方法。

🚀 安装和配置问题

问题1：安装失败 "npm install -g openclaw" 失败

症状: 安装过程中出现错误或超时

解决方案:

  1. 清理npm缓存: 
    npm cache clean --force
  2. 使用国内镜像 (中国用户): 
    npm config set registry https://registry.npmmirror.com
npm install -g openclaw
  3. 使用管理员权限 (macOS/Linux): 
    sudo npm install -g openclaw
  4. 使用nvm管理Node.js版本: 
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18
nvm use 18
npm install -g openclaw

问题2：命令找不到 "openclaw: command not found"

症状: 安装成功但无法运行openclaw命令

解决方案:

  1. 检查Node.js全局安装路径: 
    npm config get prefix
  2. 添加到PATH环境变量:
    • macOS/Linux: 在 ~/.bashrc, ~/.zshrc 或 ~/.profile 中添加: 
      export PATH="\$PATH:\$(npm config get prefix)/bin"
    • Windows: 在系统环境变量中添加Node.js安装路径
  3. 重新加载配置: 
    source ~/.zshrc  # 或 source ~/.bashrc

💬 使用和功能问题

问题3：AI助手不响应或响应慢

症状: 对话无响应或响应时间过长

解决方案:

  1. 检查模型配置: 
    # 查看当前配置
cat ~/.openclaw/openclaw.json
  2. 更换更快的模型 (如DeepSeek): 
    {
  "model": "deepseek/deepseek-chat",
  "maxTokens": 2000,
  "temperature": 0.7
}
  3. 检查网络连接 - 确保可以访问AI API
  4. 减少maxTokens - 降低响应长度限制

问题4：文件操作权限错误

症状: 无法读取或写入文件

解决方案:

  1. 检查文件权限: 
    ls -la ~/.openclaw/workspace/
  2. 修复权限: 
    sudo chown -R \$(whoami) ~/.openclaw
sudo chmod -R 755 ~/.openclaw
  3. 使用绝对路径: 
    # 而不是相对路径
openclaw chat
# 输入: 请读取 /Users/username/.openclaw/workspace/test.txt

🔌 集成和插件问题

问题5：Feishu插件配置失败

症状: Feishu集成无法正常工作

解决方案:

  1. 检查插件安装: 
    openclaw skills list
  2. 重新安装插件: 
    openclaw skills install @openclaw/feishu
  3. 检查Feishu应用配置:
    • 确保应用已发布
    • 检查权限配置
    • 验证事件订阅
  4. 查看日志: 
    tail -f /tmp/openclaw/openclaw-*.log

🛠️ 故障排除技巧

通用故障排除步骤

  1. 查看日志 - 最重要的诊断工具 
    tail -f /tmp/openclaw/openclaw-*.log
  2. 检查配置: 
    cat ~/.openclaw/openclaw.json
  3. 重启服务: 
    openclaw gateway restart
  4. 更新到最新版本: 
    npm update -g openclaw

📈 性能优化问题

问题6：内存使用过高

症状: 系统变慢，内存占用高

解决方案:

  1. 限制并发请求 - 在配置中设置
  2. 清理缓存文件: 
    rm -rf ~/.openclaw/cache/*
  3. 减少maxTokens - 降低内存使用
  4. 使用更轻量的模型

🔒 安全和权限问题

问题7：权限被拒绝错误

症状: "Permission denied" 错误

解决方案:

  1. 检查文件权限: 
    ls -la /path/to/file
  2. 修复权限: 
    sudo chown \$(whoami) /path/to/file
sudo chmod 644 /path/to/file
  3. 使用正确用户运行 - 不要使用root运行普通命令
